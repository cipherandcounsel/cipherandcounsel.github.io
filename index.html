API Reference
Threat Intelligence API · v1.0
Sources: MITRE ATT&CK · IETF RFC 7807 · Cortex XSOAR Docs View on GitHub →
Overview

The Threat Intelligence API provides programmatic access to structured threat data, adversary profiles, and IOC enrichment · enabling security teams to integrate real-time intelligence into SIEM platforms and automate alert enrichment workflows. Modeled on the Unit 42 Threat Intelligence integration published on the Cortex XSOAR Marketplace (v1.0.15, March 2026).

Endpoints
GET /indicators/{type}/{value} Single IOC Lookup

Retrieve threat intelligence for a single indicator. Returns classification, severity, confidence score, associated malware families, and MITRE ATT&CK technique mappings. Confidence scoring uses a 0–100 scale consistent with Unit 42 enrichment standards.

Parameters
typepath · string · requiredip · domain · url · hash · email
valuepath · string · requiredThe indicator value to query
include_contextquery · boolean · optionalInclude full threat actor profile. Default: false
Example Request
curl -X GET   "https://api.threatintel.example/v1/indicators/ip/198.51.100.42?include_context=true"   -H "X-API-Key: $API_KEY"
Response · 200 OK
{
  "classification": "malicious",
  "severity": "critical",
  "confidence": 94,
  "threat_types": ["command-and-control", "data-exfiltration"],
  "attack_techniques": [
    { "technique_id": "T1071.001", "tactic": "Command and Control" }
  ],
  "context": { "threat_actor": "Boggy Serpens" }
}
POST /indicators/bulk Batch Enrichment · up to 500 IOCs

Submit up to 500 indicators in a single request. Designed for high-volume SOC workflows and automated SIEM enrichment pipelines. Unknown indicators return classification: unknown rather than being omitted, ensuring no silent failures in automation.

Python · with retry logic
def bulk_lookup(indicators, max_retries=3):
    # Validate batch size per API spec
    if len(indicators) > 500:
        raise ValueError("Max 500 indicators per request")

    for attempt in range(max_retries):
        resp = requests.post(URL, json={"indicators": indicators},
                             headers={"X-API-Key": API_KEY}, timeout=30)
        if resp.status_code == 429:
            time.sleep(int(resp.headers.get("Retry-After", 60)))
            continue
        resp.raise_for_status()
        return resp.json()
GET /actors/{actor_name} Threat Actor Profile

Retrieve a detailed adversary profile including attributed campaigns, known tooling, targeted sectors, and MITRE ATT&CK technique usage. Actor data is consistent with publicly documented MITRE ATT&CK Group profiles (e.g., G0069 / MuddyWater).

SOC Playbook
SIEM Alert Enrichment Playbook · IOC Triage
Sources: NIST SP 800-61r3 · MITRE ATT&CK · Unit 42 IR Report 2026 View Document →
Purpose

This playbook defines a repeatable five-step workflow for enriching SIEM alerts with threat intelligence context and making consistent triage decisions. Structured around the NIST SP 800-61r3 Incident Response lifecycle and aligned to MITRE ATT&CK tactic tagging conventions used in Cortex XSIAM and Cortex XSOAR deployments.

Threat Context

The 2026 Unit 42 Global Incident Response Report found that attackers move from initial access to data exfiltration in under one hour in the fastest observed incidents · a 4x speed increase over prior years. Identity-based techniques drove initial access in 65% of investigated incidents. This playbook is designed for the speed and triage discipline that threat environment demands.
